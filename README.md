<p>&nbsp;<strong>Android Architecture</strong></p>
<hr>
<ol>
<li><strong>Linux Kernel</strong></li>
</ol>
<p><strong>What is a Kernel?</strong>&nbsp;The basic layer is the Linux kernel. The whole Android OS built on top of the Linux 2.6 Kernel with some further architectural changes made by&nbsp;<strong>Google</strong>. It is the core part of the&nbsp;<strong>Android&nbsp;</strong><a href="https://en.wikipedia.org/wiki/Operating_system" target="_blank" rel="nofollow noopener"><strong>Operating System</strong></a>&nbsp;that acts as an abstraction layer between the hardware and the rest of the software stack ‒ which consists of drivers (i.e., a well-defined set of instructions ‒ what we call&nbsp;<a href="https://en.wikipedia.org/wiki/Computer_program" target="_blank" rel="nofollow noopener">program</a>s or software written in&nbsp;<a href="https://en.wikipedia.org/wiki/C_(programming_language)" target="_blank" rel="nofollow noopener">C language</a>&nbsp;that installed into mobile phones and stored in the form of files in the phone) ‒ that tells your mobile phone how to communicate with its&nbsp;<strong>hardware components</strong>&nbsp;such as camera, display etc. ‒ without which keypad,&nbsp;<a href="https://en.wikipedia.org/wiki/Bluetooth" target="_blank" rel="nofollow noopener">Bluetooth</a>, Audio,&nbsp;<a href="https://en.wikipedia.org/wiki/Wi-Fi" target="_blank" rel="nofollow noopener"><strong>Wi-Fi</strong></a>, Camera won't work properly and it is responsible for:</p>
<ul>
<li><strong>Inter Process Communication -</strong>&nbsp;A Mechanism which allows applications running in different processes to share data and communicate with each other i.e., a mechanism which allows an application running in a process to send requests and receive responses from an application running in another process.</li>
<li><strong>Power Management</strong>&nbsp;(conserves power in the cost of performance and holds the device not to get to sleep state).</li>
<li><strong>Memory Management</strong>&nbsp;(make the best or most effective use of&nbsp;<a href="https://en.wikipedia.org/wiki/Memory" target="_blank" rel="nofollow noopener">memory</a>).</li>
</ul>
<p>&nbsp;Android uses the Linux Kernel for all its core functionality such as Memory management, process management,&nbsp;<a href="https://en.wikipedia.org/wiki/Computer_network" target="_blank" rel="nofollow noopener">network</a>ing,&nbsp;<a href="https://en.wikipedia.org/wiki/Security" target="_blank" rel="nofollow noopener">security</a>&nbsp;settings etc.</p>
<p>&nbsp;</p>
<p><strong>2. Libraries</strong></p>
<p>The next layer is the Android's native libraries. It is this layer that enables the device to handle different types of data. These libraries are a Collection of&nbsp;<strong>pre-written</strong>&nbsp;non-volatile&nbsp;<a href="https://en.wikipedia.org/wiki/Data" target="_blank" rel="nofollow noopener">data</a>&nbsp;(written in C /&nbsp;<a href="https://en.wikipedia.org/wiki/C%2B%2B" target="_blank" rel="nofollow noopener">C++ language</a>) and pre-compiled programming codes ‒ which support the well-functioning of android operating system.</p>
<p><strong>Some of the important native libraries include the following:</strong></p>
<ul>
<li>&nbsp;Surface Manager / Screen Manager that supports the display screen.</li>
<li>OpenGL (<strong>Open Graphics Library</strong>) that supports 3 dimensional&nbsp;<a href="https://en.wikipedia.org/wiki/Graphics" target="_blank" rel="nofollow noopener">graphics</a>.</li>
<li>SGL (<strong>Scalable Graphics Library</strong>) that supports 2 dimensional graphics.</li>
<li>Media Framework that supports recording and playback of audio and video and image formats (MP3, JPG, JPEG,&nbsp;<strong>PNG</strong>, GIF etc.)</li>
<li>Free Type that is responsible for&nbsp;<a href="https://en.wikipedia.org/wiki/Font" target="_blank" rel="nofollow noopener"><strong>font</strong></a><strong>&nbsp;support</strong>&nbsp;(i.e., font size, color etc.)</li>
<li><strong>SSL</strong>&nbsp;(Secured Sockets layer) /&nbsp;<strong>TLS</strong>&nbsp;(Transport Layer Security) that is responsible for internet security and support network applications.</li>
<li><strong>WebKit&nbsp;</strong>that supports the display of&nbsp;<a href="https://en.wikipedia.org/wiki/World_Wide_Web" target="_blank" rel="nofollow noopener">web</a>&nbsp;pages (i.e., supports inbuilt&nbsp;<a href="https://en.wikipedia.org/wiki/Web_browser" target="_blank" rel="nofollow noopener">browser</a>)</li>
<li><strong>SQLite</strong>&nbsp;that is responsible for storage of user data.</li>
<li><strong>Bionic&nbsp;</strong>is the standard C library that supports embedded&nbsp;<a href="https://en.wikipedia.org/wiki/Linux" target="_blank" rel="nofollow noopener">Linux</a>-based devices in mobile phones.</li>
</ul>
<p>&nbsp;</p>
<p><strong>&nbsp;3</strong>.&nbsp;<strong>Android Run Time (ART)</strong></p>
<p>Android Runtime consists of Core Java libraries and&nbsp;<a href="https://en.wikipedia.org/wiki/Dalvik_(software)" target="_blank" rel="nofollow noopener"><strong>Dalvik&nbsp;</strong></a>Virtual machine.</p>
<ul>
<li>Java Core Libraries that consists of Java packages that enable Android application developers to write Android applications using&nbsp;<strong>standard Java programming language</strong>.</li>
<li>DVM (<strong>Dalvik&nbsp;</strong><a href="https://en.wikipedia.org/wiki/Virtual_machine" target="_blank" rel="nofollow noopener"><strong>Virtual Machine</strong></a>) that is responsible to run android application.</li>
</ul>
<p>&nbsp;</p>
<p>&nbsp;<strong>4. Application Frame Work</strong></p>
<p><strong>Software Frame work</strong>&nbsp;(written in Java language) that supports the features of android applications and manage the basic functions of phone like resource management, voice call management etc.</p>
<p><strong>Important blocks of Application framework are:</strong></p>
<ul>
<li><strong>Content Provider</strong>&nbsp;that enable applications to get access data from other applications (such as Contacts), or to share their own data.</li>
<li><strong>Notifications Manager</strong>&nbsp;that enables all applications to display custom alerts in the status bar.</li>
<li><strong>Activity Manager</strong>&nbsp;that manages the life-cycle of applications and provides a common navigation back stack.</li>
<li><strong>Window Manager</strong>&nbsp;that organizes the display screen for the application.</li>
<li><strong>Location Manager</strong>&nbsp;that provides the periodic updates of the geographical location of the mobile device using&nbsp;<strong>GPS</strong>&nbsp;(<a href="https://en.wikipedia.org/wiki/Global_Positioning_System" target="_blank" rel="nofollow noopener">Global Positioning System</a>&nbsp;which is a&nbsp;<a href="https://en.wikipedia.org/wiki/Satellite" target="_blank" rel="nofollow noopener">satellite</a>-based navigation system) or cell tower.</li>
<li><strong>View Manager</strong>&nbsp;that manages the Application User Interface.</li>
<li><strong>Package Manager</strong>&nbsp;that provides information about the list of installed apps in Android&nbsp;<a href="https://en.wikipedia.org/wiki/Mobile_device" target="_blank" rel="nofollow noopener">Mobile Device</a>.</li>
<li><strong>Telephony Manager</strong>&nbsp;that provides information about the&nbsp;<strong>Telephony Services</strong>&nbsp;(such as Phone Network, SIM Serial Number,&nbsp;<strong>IMEI Number</strong>&nbsp;etc.)</li>
<li>&nbsp;<strong>XMPP</strong>&nbsp;(Extensible Messaging and Presence Protocol) that supports Online Chat Application (like&nbsp;<a href="https://en.wikipedia.org/wiki/Yahoo!_Messenger" target="_blank" rel="nofollow noopener">Yahoo Messenger</a>&nbsp;etc.)</li>
<li><strong>Resource Manager</strong>&nbsp;that manages the various types of resources we use in our Application and provides access to&nbsp;<strong>non-code resources</strong>&nbsp;such as localized strings, graphics, and layout files.</li>
</ul>
